# 📡 API Testing Notes — Intermediate

> A hands-on reference for testers who know the basics and want to go deeper.

---

## 🗂 What's inside

This project is a single-file interactive HTML notes page covering everything an intermediate API tester needs to know — with charts, checklists, and a quiz built in.

| Section | What you'll learn |
|---|---|
| Core Concepts | Request & response anatomy, testing types |
| HTTP Methods | GET, POST, PUT, PATCH, DELETE — what to verify for each |
| Status Codes | 200, 300, 400, 500 with common gotchas called out |
| Auth & Security | API keys, JWTs, OAuth 2.0, BOLA/IDOR, rate limiting |
| Advanced Testing | Contract, performance, negative testing, chaining requests |
| Quick Quiz | 7 questions with instant feedback and a score tracker |

---

## 🚀 How to use

**Option 1 — Open locally**
Download `api-testing-notes.html` and open it in any browser. No server needed.

**Option 2 — GitHub Pages (live website)**
1. Go to your repo **Settings → Pages**
2. Set source to `main` branch
3. Your notes will be live at:
   `https://<your-username>.github.io/<your-repo>/api-testing-notes.html`

---

## ✅ Prerequisites

You should already be comfortable with:
- What an API is and how to send a basic request
- The difference between a client and a server
- Basic JSON structure

No prior knowledge of security testing or performance testing needed — those are covered inside.

## 🛠 Built with

- Plain HTML, CSS, and JavaScript — no frameworks, no build step
- [Chart.js](https://www.chartjs.org/) for the charts
- Works in all modern browsers, supports dark mode
